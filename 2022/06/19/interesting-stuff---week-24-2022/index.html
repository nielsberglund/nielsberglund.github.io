Interesting Stuff - Week 24, 2022

Jun 19, 2022 in ROUNDUP
ai distributed computing sql server azure data studio azure data explorer kafka
2 min read

Performance Photo by Todd Quackenbush on Unsplash

Throughout the week, I read a lot of blog-posts, articles, and so forth that has to do with things that interest me:

  • AI/data science
  • data in general
  • data architecture
  • streaming
  • distributed computing
  • SQL Server
  • transactions (both db as well as non db)
  • and other “stuff”

This blog post is the “roundup” of the things that have been most interesting to me for the week just ending.

Misc.

  • Making Code Faster. This post is a continuation of the post here, which I covered in the roundup for week 19, 2022. The post discusses various approaches to performance testing of your code. Very interesting!

Machine Learning/AI

  • Distributed Computing for AI: A Status Report. The post linked to looks at what role distributed computing plays in AI. The consensus is that it becomes more and more essential to understand distributed computing in AI projects. Also, when hiring for AI, you often compete with “pure” distributed computing roles.

SQL Server

  • Recapping recent updates to client tools in SQL Server. It has been a while since I have covered tools for SQL Server. Let me rectify that with this link. As the title implies, this post looks at SQL Server tools. What caught my eye was the notebook enhancements in Azure Data Studio and the containerization of Azure SQL Database! Cool stuff!

Azure Data Explorer

  • 5 Ways in which ADX can provide a leading edge to your Data Platform. I am pretty “fond” of Azure Data Explorer as you know if you read my blog. I particularly like the super-fast ingestion and querying and the ability to do real-time analysis! However, there are other advantages of ADX, and the post I have linked to looks at 5 of those.

Streaming

  • How Confluent Treats Incidents in the Cloud. Since we at Derivco looks at using Confluent Cloud, this post came in real handy. As the title says, the post looks at Confluent Cloud incidents and how they are handled. Very informative!

~ Finally

That’s all for this week. I hope you enjoy what I did put together. Please comment on this post or ping me if you have ideas for what to cover.
